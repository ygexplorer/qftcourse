# 第二版建设路线图

> 基于第一版上线后（2026-03-29）的全面代码审计，结合网站每学期重复使用的实际需求。

---

## 一、当前核心能力盘点

### 已完整实现

| 模块 | 能力 | 涉及角色 |
|------|------|----------|
| **用户系统** | 注册/登录/改密/角色控制/权限中间件/批量导入 | 学生自行注册或批量导入，TA/教师由管理员创建 |
| **课程内容** | 学期管理 → 章节列表 → 章节详情（Markdown 渲染、上下章导航） | 教师草稿预览，TA/学生只看已发布 |
| **作业系统** | 发布作业（含 PDF 附件）→ 学生提交（PDF、逾期标记）→ 评分/评语 → 下载 | 完整的逾期检测、文件校验、权限控制、评分闭环 |
| **公告系统** | 发布公告 → 学期公告列表（置顶）→ 公告详情（Markdown） | 不需登录即可浏览 |
| **评分工作台** | 卡片式提交列表 → 内联评分 → 筛选/搜索 → 保存并下一个 | 教师/TA 使用，学生不可访问 |
| **管理后台** | Django Admin 管理：学期/章节/作业/提交/用户/公告 | 教师 Admin 权限自动同步 |
| **部署** | 腾讯云 HTTPS、PostgreSQL、Gunicorn、systemd、Nginx | — |

## 二、功能优先级排序

### 第一优先 — ✅ 全部完成

#### ~~1. 📢 公告系统前端~~ → ✅ 第七阶段已完成

| 维度 | 说明 |
|------|------|
| **为什么做** | 模型和 Admin 已完整实现，前端零成本接入。每学期都需要发布通知（调课、考试安排等），当前完全无法触达学生 |
| **价值** | 教师 → 发布公告通知全班；学生/TA → 首页/学期页看到最新公告 |
| **复杂度** | 🟢 低 — 新增 1 个视图 + 1 个模板，约 100 行代码 |
| **适合现在** | ✅ 立刻开始，最短时间获得最高实用价值 |

实现要点：
- 学期首页顶部显示该学期公告列表（最新 5 条）
- 置顶公告始终在顶部
- 公告详情页（支持 Markdown 渲染复用 `{{ content|markdown }}`）
- 导航栏添加"公告"入口（带未读标记可选，第二版不做也行）

---

#### ~~2. ✏️ 在线评分与评语~~ → ✅ 第八阶段已完成

| 维度 | 说明 |
|------|------|
| **为什么做** | `score` 和 `feedback` 字段已在数据库和 Admin 中就位，但 TA/教师在前台完全无法评分。作业系统的闭环没有完成 |
| **价值** | TA/教师 → 在提交列表直接打分、写评语；学生 → 查看自己的分数和评语 |
| **复杂度** | 🟡 中低 — 修改 assignment_detail 模板 + 新增评分表单/视图，约 200 行 |
| **适合现在** | ✅ 评分是作业系统的核心闭环，不做的话作业提交只是"扔进黑箱" |

实现要点：
- TA/教师查看提交时，每行添加"评分"和"评语"输入框（可展开/折叠）
- 支持在提交列表页批量保存（AJAX 或传统表单都行）
- 学生查看自己的提交时，显示分数和评语
- 评分可选范围（如 0-100）或自由输入，视你的需求定

---

#### ~~3. 👤 学生个人资料编辑~~ → ✅ 第十一阶段已完成

| 维度 | 说明 |
|------|------|
| **为什么做** | 学生注册时可能没填全信息，但之后无法修改姓名和学号。每学期都可能需要更新 |
| **价值** | 学生 → 修改显示名称、学号 |
| **复杂度** | 🟢 低 — 新增 1 个视图 + 1 个模板，约 80 行代码 |
| **已完成** | ✅ 第十一阶段 — `ProfileEditForm` + `profile_edit` 视图 + 模板 + URL + 导航栏入口 |

---

#### ~~4. 📋 批量导入学生~~ → ✅ 第十阶段已完成

| 维度 | 说明 |
|------|------|
| **为什么做** | 每学期开学，教师需要创建几十个学生账号。逐个在 Admin 中添加太痛苦 |
| **价值** | 教师 → 上传 Excel/CSV，一键批量创建学生账号 |
| **复杂度** | 🟡 中 — 需要处理 CSV 解析、去重、密码生成、错误提示，约 300 行 |
| **适合现在** | ✅ 这是每学期必然要做的操作，手动创建学生不现实 |

实现要点：
- 教师在仪表盘上传 CSV 文件（格式：username, display_name, student_id）
- 自动生成默认密码（如学号后 6 位或随机密码）
- 显示导入结果（成功/失败/跳过（用户名已存在））
- 可选：生成可打印的账号密码列表

### 第二优先 — 上线稳定后再做

#### 5. 📊 教师仪表盘动态数据

| 维度 | 说明 |
|------|------|
| **为什么做** | 教师登录后看到的是静态链接页，无法快速掌握班级状态 |
| **价值** | 教师 → 一目了然：待批改数量、未提交学生、最近活动 |
| **复杂度** | 🟡 中 — 需要 SQL 查询聚合 + 模板渲染 |
| **适合现在** | ⏸️ 等评分系统上线后再做，否则"待批改数"无法准确显示 |

#### 6. 📂 课程资料/参考材料

| 维度 | 说明 |
|------|------|
| **为什么做** | 教师除了章节内容和作业附件，还可能想分享讲义、参考文献、往年试卷等 |
| **价值** | 所有角色 → 浏览和下载课程相关资料 |
| **复杂度** | 🟡 中 — 需要新建 Resource 模型 + 上传/下载 + 列表/详情页 |
| **适合现在** | ⏸️ 不紧急，可以先通过章节内容内嵌链接解决 |

#### 7. 📝 提交历史记录

| 维度 | 说明 |
|------|------|
| **为什么做** | 当前重新提交直接覆盖旧记录，学生和教师都看不到修改历史 |
| **价值** | 教师/TA → 查看学生的提交演变；学生 → 确认自己的提交被正确记录 |
| **复杂度** | 🔴 高 — 需要修改数据模型（去掉 unique_together 或建新表）、迁移已有数据 |
| **适合现在** | ⏸️ 涉及数据库结构变更，建议稳定运行一学期后再考虑 |

---

### 远期功能 — 先不做

| 功能 | 说明 | 不做原因 |
|------|------|----------|
| **通知系统** | 新作业/评分/公告的邮件或站内通知 | 第一版学生量不大，当面通知或微信群足够 |
| **在线 Markdown 编辑器** | 教师在前台编辑章节而非 Django Admin | Django Admin 已经可用，独立编辑器是锦上添花 |
| **文件存储迁移 COS** | 把上传文件从服务器迁移到腾讯云 COS | 当前服务器空间够用，量不大时没必要增加复杂度 |
| **单元测试** | 自动化测试覆盖核心功能 | 对单人维护项目来说 ROI 不高，等稳定后可补 |
| **深色模式** | 暗色主题 | 不是核心功能，优先级最低 |
| **多语言/国际化** | 英文版 | 课程面向国内学生，暂不需要 |

## 三、建议的下一阶段开发顺序

### 第七～十阶段：教学实用化（已完成 ✅）

已完成功能按顺序：

```
✅ 公告系统（第七阶段） → ✅ 评分评语（第八阶段） → ✅ 评分工作台（第九阶段） → ✅ 批量导入学生（第十阶段） → ✅ 学生资料编辑（第十一阶段） → ✅ 章节导航与讲义下载（第十二阶段）
```

**成果**：
1. **公告系统**（第七阶段）—— 教师可发布通知，学生无需登录即可查看
2. **评分与评语**（第八阶段）—— TA/教师可打分（0-100）写评语，学生可查看成绩
3. **评分工作台**（第九阶段）—— 统一评分界面，卡片式列表，筛选搜索，"保存并下一个"流程
4. **批量导入学生**（第十阶段）—— Admin 上传 Excel，一键创建学生账号
5. **学生资料编辑**（第十一阶段）—— 登录用户可修改 display_name 和 student_id
6. **章节导航与讲义下载**（第十二阶段）—— 左侧目录、子标题锚点、IntersectionObserver 滚动高亮、讲义 PDF 下载

### 第十一阶段及以后：增强体验

```
✅ 学生资料编辑 → ✅ 章节导航与讲义下载 → 仪表盘数据 → 课程资料 → 提交历史 → 通知系统
```

## 四、代码健康度备注

以下是审计中发现的非功能性改进点，可以随时顺手修：

1. ~~**`courses/decorators.py` 冗余** — 与 `accounts/decorators.py` 功能重复，`views.py` 实际导入的是 accounts 版本。可安全删除 `courses/decorators.py`~~ → ✅ 已删除
2. **无自定义静态文件** — 所有 CSS 内联在模板中。如果 Markdown 样式越来越长，建议抽取到 `static/css/` 文件
3. **Tailwind 使用 CDN** — `cdn.tailwindcss.com` 是开发版本，生产环境建议换为编译版本（当前量级下不紧急）
4. **单元测试缺失** — `tests.py` 均为空，核心功能没有自动化测试覆盖
