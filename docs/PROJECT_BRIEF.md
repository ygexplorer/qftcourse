# 规范场课程网站 — 项目简介

## 项目名称

规范场课程（Gauge Field Theory Course Website）

## 项目目标

为理论物理课程"规范场"提供一个轻量的教学辅助网站，支持课程内容发布和学生作业管理。第一版侧重核心功能，不做复杂的在线评分和历史记录系统。

## 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 后端框架 | Django 5.1.7 | Python 3.12 |
| 数据库 | PostgreSQL（生产）/ SQLite（开发） | Django ORM |
| 前端样式 | Tailwind CSS | CDN 引入，无需构建 |
| 交互 | Alpine.js | CDN 引入，轻量交互 |
| 内容渲染 | python-markdown 3.10.2 | 章节内容使用 Markdown 编写 |
| 部署 | 腾讯云 CVM（Ubuntu 22.04） | Nginx + Gunicorn + systemd |

## 用户角色

| 角色 | 创建方式 | 核心功能 |
|------|----------|----------|
| student | 自行注册 | 浏览课程内容、提交作业、查看自己的提交 |
| ta | 教师在 Admin 中创建 | 查看所有学生提交、下载文件、查看作业附件 |
| teacher | 管理员创建 | 管理课程内容（Admin）、发布作业、查看所有提交 |
| admin | 命令行 createsuperuser | 全部权限 |

## 核心功能

### 课程内容管理
- 学期管理（支持多个学期，标记当前学期）
- 章节管理（Markdown 编写，支持草稿/发布状态）
- 章节支持有序排列和上下章导航

### 作业系统
- 教师通过 Django Admin 发布作业，可附 PDF 附件
- 学生前台提交 PDF 文件（20MB 大小限制）
- 支持逾期提交（必须填写逾期原因，系统标记逾期状态）
- 重新提交覆盖旧文件，不保留提交历史
- 作业列表清晰显示提交状态（未提交 / 已提交 / 逾期提交 / 已逾期未提交）

### 文件管理
- 学生提交文件自动命名：`username_a{作业ID}_{日期时间}.pdf`
- 教师附件自动命名：`attachment_a{作业ID}_{日期时间}.pdf`
- 原始文件名保存在数据库，下载时恢复原始文件名
- 按类型/年/月分目录存储，方便以后迁移至对象存储

## 第一版不包含

- 在线评分系统（模型字段已预留，前台不展示）
- 提交历史记录（重新提交直接覆盖）
- 公告系统前台展示（模型已有，第一版不启用）
- 在线 Markdown 编辑器
- 前端构建工具
